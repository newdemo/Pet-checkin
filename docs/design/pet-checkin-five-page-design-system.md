# 宠物打卡完整五页 UI 方案

## 设计母版

方向：宠物家园首屏。

产品第一印象必须是“我的宠物在等我”，不是“今天有几个任务”。任务、金币、成长值都服务于宠物照顾、家园建设和内容解锁。

核心体验：

宠物陪伴 -> 照顾任务 -> 获得奖励 -> 宠物成长 -> 解锁家具/皮肤/家园

## 视觉系统

### 视觉气质

- 现代移动 App 风格
- Apple 感的大留白和柔和层级
- 高品质 3D 宠物主视觉
- 温暖、干净、轻盈
- 大圆角、低噪音、少装饰
- 儿童喜欢，但不幼稚

### 页面底色

- 主背景：暖奶油色
- 内容面：柔白色
- 强调色：珊瑚橙、薄荷绿、柔金色
- 状态色：饱腹用橙，清洁用薄荷，心情用黄，成长用暖金

### 圆角与层级

- 主容器：28-36px 圆角
- 行动按钮：999px 胶囊圆角
- 小状态芯片：16-22px 圆角
- 阴影：只用于宠物主舞台、浮层和重点奖励，不用于每个列表项

### 字体层级

- 页面标题：22-28px，粗体
- 宠物名/阶段：24-32px，粗体
- 主要按钮：17-20px，半粗体
- 正文/说明：14-16px
- 状态小字：12-13px

### 导航

底部保留五个 Tab：

- 首页
- 宠物
- 任务
- 成长
- 我的

Tab 文案可以保留“任务”，但页面内容表达应使用“今日照顾”，避免任务管理感。

## 1. 首页

### 页面定位

首页是宠物家园，不是任务入口。

### 首屏结构

顶部：

- 小梨 Lv.3
- 金币
- 心情状态
- 家长入口弱化为小图标或右上角次级入口

主视觉：

- 小梨占据首屏 50% 以上视觉面积
- 背景是柔和家园空间
- 展示 1-2 个可解锁家具轮廓
- 宠物气泡表达情绪，例如：“小梨想和你一起布置新家”

主要操作：

- 喂食
- 清洁
- 陪玩

次级模块：

- 今日照顾任务
- 展示 1/5 进度
- 只露出 2 条任务
- 任务奖励直接关联宠物成长，例如食物、成长值、家具星星

### 首页关键规则

孩子打开首页时，应先想点宠物，而不是先找任务。

## 2. 宠物页

### 页面定位

宠物页是“小梨的家”，用于查看宠物状态、情绪、家园布置和照顾动作。

### 页面结构

顶部：

- 标题：小梨的家
- 状态芯片：Lv.3、幼年期、心情很好

主视觉：

- 小梨位于家园中央
- 周围展示家具/装饰位
- 宠物气泡表达当前需求，例如：“今天想换个小窝”

状态区：

- 饱腹
- 清洁
- 心情
- 成长值

状态条必须紧凑，不能压过宠物主视觉。

家园布置：

- 小窝
- 灯
- 玩具角
- 花园角

每个模块呈现已解锁/未解锁状态，强化建设感。

底部操作：

- 喂食
- 洗澡
- 陪玩

### 宠物页关键规则

数值只是解释宠物状态，宠物表情和家园变化才是反馈主体。

## 3. 任务页

### 页面定位

任务页应表达为“今日照顾”，不是待办清单。

### 页面结构

顶部：

- 标题：今日照顾
- 小梨小头像或家园角落
- 提示语：“完成照顾，小梨会长大一点”
- 今日进度：1/5

任务卡：

每条任务包含：

- 任务图标或小插画
- 任务名称
- 宠物奖励
- 状态按钮

示例：

- 刷牙冒险：+3 食物，+5 成长值
- 英语故事：+8 成长值，+1 家具星星
- 运动能量：+3 玩具，+6 心情
- 探索世界：+10 成长值，+1 金币宝箱

按钮状态：

- 去完成
- 等爸爸确认
- 已奖励

底部奖励预告：

- 下一次解锁：小星灯
- 进度：62/80

### 任务页关键规则

任务不是管理孩子，而是帮助孩子照顾宠物和解锁家园。

## 4. 成长页

### 页面定位

成长页是成长路线和收藏图鉴，不是打卡历史记录。

### 页面结构

顶部：

- 小梨头像或半身主视觉
- 状态语：“被照顾得很开心”
- 阶段：Lv.3 幼年期

成长路线：

- Lv.3 幼年期
- Lv.4 探索期
- Lv.5 换装解锁
- Lv.6 家园扩建

下一阶段需要有明显发光或奖励预告。

下一次解锁：

- 奖励名：小星灯
- 类型：家具
- 条件：成长值 80
- 当前：62/80
- 行动：继续照顾

收藏分类：

- 家具 3/12
- 皮肤 1/8
- 玩具 4/16
- 小窝 1/6

最近获得：

- 只作为次级记录
- 不再占据页面核心

### 成长页关键规则

孩子进入成长页时，应看到“我快解锁什么了”，不是“我过去完成了什么”。

## 5. 我的页面

### 页面定位

我的页面是“我的宠物档案”和家庭账户，不是设置页。

### 页面结构

顶部档案：

- 小梨的小主人
- 小梨主视觉
- 陪伴天数
- 家园等级
- 收集进度
- 金币

宠物档案：

- 名称：小梨
- 阶段：Lv.3 幼年期
- 当前心情：开心
- 最喜欢：陪玩
- 领养日
- 编辑宠物名入口

我的收藏：

- 家具
- 皮肤
- 玩具
- 小窝

家长区域：

- 家长管理
- 待确认 1
- 任务设置

家长入口必须安静、清晰，但不成为孩子视角主入口。

设置区域：

- 通知
- 声音
- 数据备份

### 我的页面关键规则

我的页首先是孩子与宠物关系的档案，其次才是账户和设置。

## 组件系统

### 宠物主舞台

用于首页和宠物页。

包含：

- 宠物 3D 主视觉
- 柔和家园背景
- 情绪气泡
- 可解锁家具提示

### 状态芯片

用于顶部与宠物状态：

- Lv.3
- 心情很好
- 金币 860
- 家园 Lv.2

### 照顾按钮

三枚大胶囊按钮：

- 喂食
- 清洁/洗澡
- 陪玩

按钮要像“和宠物互动”，不要像“提交任务”。

### 今日照顾卡

任务卡必须包含宠物奖励，弱化任务管理语气。

建议结构：

任务名 -> 奖励 -> 状态 -> 行动

### 解锁预告卡

用于首页、任务页、成长页。

包含：

- 奖励图片
- 奖励名称
- 解锁条件
- 当前进度
- 继续照顾按钮

## 文案方向

推荐：

- 小梨想你啦
- 今天想换个小窝
- 完成照顾，小梨会长大一点
- 再照顾 2 次就能解锁小星灯
- 小梨被照顾得很开心

避免：

- 今日任务列表
- 待办事项
- 管理任务
- 完成打卡
- 数据统计

## 实现优先级

第一优先级：

- 首页宠物主视觉
- 今日照顾托盘
- 三个照顾动作

第二优先级：

- 宠物页家园布置
- 成长页下一次解锁
- 任务页奖励绑定

第三优先级：

- 收藏图鉴
- 皮肤与家具解锁
- 我的宠物档案

## 总结

这套五页方案保持方案 1 的核心：宠物是主角，任务是照顾宠物的手段。页面从“任务、统计、管理”转向“宠物、家园、成长、解锁”，更符合 5-10 岁儿童的情绪驱动和长期留存。
