# P4-2A 最小数据库集合设计（P4 执行版）

## 1. 文档定位

本文档是 P4 阶段云数据库设计的唯一执行依据。

P4-2B 只允许依据本文档创建集合，不得依据旧版 `docs/数据库设计.md` 中的历史集合、snake_case 字段或旧状态流转创建数据库结构。

## 6. 集合设计

### 6.1 pet

**用途：** 存储孩子对应的宠物状态、成长数据和扩展信息。

| 字段名 | 字段类型 | 是否必填 | 默认值 | 字段说明 |
|--------|----------|----------|--------|----------|
| `_id` | string | 自动 | 云数据库自动生成 | 文档 ID |
| `openid` | string | 是 | 无 | 微信用户 openid |
| `familyId` | string | 是 | `family_${openid}` | 家庭 ID |
| `childId` | string | 是 | `child_default_${openid}` | 孩子 ID |
| `name` | string | 是 | `小萌` | 宠物名称 |
| `level` | number | 是 | `1` | 宠物等级 |
| `exp` | number | 是 | `0` | 成长值 |
| `hunger` | number | 是 | `80` | 饥饿值 |
| `cleanliness` | number | 是 | `80` | 清洁值 |
| `mood` | number | 是 | `80` | 心情值 |
| `avatarStage` | number | 否 | `1` | 宠物形象阶段 |
| `statusVersion` | number | 否 | `1` | 状态模型版本 |
| `createdAt` | date | 是 | 服务端时间 | 创建时间 |

## 7. 索引设计

| 集合 | 建议索引 | 类型 | 用途 |
|------|----------|------|------|
| `pet` | `openid + familyId + childId` | 唯一组合索引 | 查询并约束当前孩子唯一宠物 |
| `inventory` | `openid + familyId + childId` | 唯一组合索引 | 查询并约束当前孩子唯一背包 |
| `taskTemplates` | `openid + familyId + childId + enabled + sortOrder` | 普通组合索引 | 查询启用模板并排序 |
| `dailyTasks` | `openid + familyId + childId + taskDate` | 普通组合索引 | 查询某个孩子当日任务列表 |
| `dailyTasks` | `openid + familyId + childId + taskDate + templateId` | 唯一组合索引 | 防止同一模板同一天重复生成 |
| `checkinRecords` | `dailyTaskId + status` | 普通组合索引 | 查询任务关联打卡状态 |
| `checkinRecords` | `openid + familyId + childId + status` | 普通组合索引 | 查询待审核列表 |

P4-2A 仅输出索引设计，不创建索引。索引创建属于 P4-2B 或后续数据库配置步骤。

## 9. 状态流转设计

### 9.1 dailyTasks.status

P4 执行版只允许以下状态：

| 状态值 | 含义 | 下一步 |
|--------|------|--------|
| `pending` | 待完成，孩子尚未提交 | 可提交打卡 |
| `submitted` | 已提交，等待家长确认 | 可通过或驳回 |
| `approved` | 家长已确认 | 流程结束 |

### 9.2 checkinRecords.status

P4 执行版只允许以下状态：

| 状态值 | 含义 | 下一步 |
|--------|------|--------|
| `submitted` | 孩子已提交，等待家长审核 | 可通过或驳回 |
| `approved` | 家长确认通过 | 流程结束 |
| `rejected` | 家长驳回 | dailyTasks 回到 pending |

### 9.3 驳回规则

家长驳回时：

- `checkinRecords.status = rejected`
- `checkinRecords.rewardGranted = false`
- `dailyTasks.status = pending`
- `dailyTasks.checkinRecordId = null`
- `dailyTasks.rejectedAt = 服务端时间`
- 不发放奖励

## 10. 防重复发奖设计

`checkinRecords.rewardGranted` 默认值为 `false`。

发奖前必须校验：

- `dailyTasks.status === 'submitted'`
- `checkinRecords.status === 'submitted'`
- `checkinRecords.rewardGranted === false`

确认通过后必须完成：

- 增加 `inventory[rewardType]`
- 更新 `dailyTasks.status = approved`
- 更新 `checkinRecords.status = approved`
- 更新 `checkinRecords.rewardGranted = true`

任一校验失败，不允许发奖。

## 11. P4 云函数边界

P4 阶段只保留以下云函数：

| 云函数 | P4 是否进入 | 用途 |
