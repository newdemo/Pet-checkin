# 萌宠打卡小程序 — MVP 产品需求文档

| 项目 | 内容 |
|------|------|
| 项目名称 | 萌宠打卡小程序 |
| 版本 | MVP v1.0 |
| 目标用户 | 5 岁左右儿童及家长 |
| 技术栈 | UniApp + 微信小程序；后端优先微信云开发 / 本地模拟数据 |
| 文档日期 | 2026-06-08 |

---

## 1. 产品概述

### 1.1 产品定位

萌宠打卡是一款面向学龄前儿童的**习惯养成 + 电子宠物养成**微信小程序。家长设置每日生活/学习任务，孩子完成任务后打卡，经家长确认后获得道具奖励，用于喂养、清洁、陪玩虚拟宠物，推动宠物成长升级。

### 1.2 产品目标

- 帮助孩子建立刷牙、阅读、收拾玩具、运动、练字等日常习惯。
- 通过即时、可视化的宠物反馈，增强孩子完成任务的动机。
- 让家长低成本配置任务、确认打卡、查看成长记录。

### 1.3 MVP 范围说明

**包含：**

- 宠物首页（状态展示）
- 今日任务与打卡
- 家长确认打卡
- 道具奖励与宠物养成
- 家长任务管理
- 成长记录

**不包含（后续版本）：**

- 支付、会员
- 社交、分享裂变
- 排行榜
- 复杂动画、3D 宠物
- 多宠物、多孩子账号体系（MVP 默认一个家庭一个宠物）

---

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|----------|
| 孩子 | 主要使用者，约 5 岁 | 简单操作、即时反馈、有趣可爱 |
| 家长 | 任务配置者与确认者 | 灵活设置任务、确认真实性、查看进度 |

### 2.1 角色切换

MVP 采用**同一微信账号下的角色切换**（首页顶部或设置页切换「孩子模式 / 家长模式」），无需独立登录体系。家长模式进入时需简单验证（如点击「我是家长」+ 确认弹窗），防止孩子误操作任务设置。

---

## 3. 核心用户流程

```
家长设置每日任务
    ↓
孩子查看今日任务 → 完成任务 → 点击「打卡」
    ↓
家长收到待确认 → 确认通过
    ↓
系统发放道具奖励（食物 / 肥皂 / 玩具）
    ↓
孩子在宠物页使用道具 → 提升饥饿 / 清洁 / 心情值 + 成长值
    ↓
成长值累计 → 宠物升级 → 成长记录更新
```

## 4. 功能需求

### 4.1 宠物首页

**功能描述：** 展示当前电子宠物及核心状态，是孩子进入小程序后的主界面。

**展示内容：**

| 字段 | 说明 | 数值范围 |
|------|------|----------|
| 宠物形象 | 根据等级展示不同静态形象（图片即可） | Lv.1 ~ Lv.5 |
| 宠物名称 | 默认「小萌」，可在家长设置中修改 | 文本 |
| 成长等级 | Lv.1 ~ Lv.5 | 整数 |
| 成长值 | 当前等级进度条 | 0 ~ 升级阈值 |
| 饥饿值 | 喂食道具提升 | 0 ~ 100 |
| 清洁值 | 肥皂道具提升 | 0 ~ 100 |
| 心情值 | 玩具道具提升 | 0 ~ 100 |
| 道具背包 | 食物、肥皂、玩具数量简要展示 | 整数 |

**交互：**

- 点击「喂食」「洗澡」「陪玩」按钮，消耗对应道具并提升状态值。
- 道具不足时，按钮置灰并提示「完成任务可获得更多道具」。
- 状态值随时间缓慢下降（MVP 可设为每日重置或固定初始值，见技术说明）。

**验收标准：**

- [ ] 进入首页可看到宠物及四项核心数值
- [ ] 使用道具后数值实时更新
- [ ] 成长值满后自动升级，等级 +1

---

### 4.2 今日任务

**功能描述：** 展示家长为当天配置的任务列表，供孩子查看与打卡。

**任务字段：**

| 字段 | 说明 |
|------|------|
| 任务名称 | 如：刷牙、阅读、收拾玩具 |
| 任务图标 | 预设图标库选择 |
| 奖励类型 | 食物 / 肥皂 / 玩具 |
| 奖励数量 | 默认 1，家长可配置 1~3 |
| 状态 | 未完成 / 待确认 / 已完成 |

**预设任务模板（家长可一键添加）：**

- 刷牙 → 奖励肥皂 ×1
- 阅读 15 分钟 → 奖励食物 ×1
- 收拾玩具 → 奖励玩具 ×1
- 运动 20 分钟 → 奖励食物 ×1
- 练字 → 奖励食物 ×1

**交互：**

- 孩子点击「打卡」→ 状态变为「待确认」，不可重复打卡。
- 家长确认后 → 状态变为「已完成」，道具入账。
- 家长驳回（可选）→ 状态回退为「未完成」，孩子可重新打卡。

**验收标准：**

- [ ] 当日任务列表正确展示
- [ ] 打卡后状态变更，待确认任务在家长端可见
- [ ] 确认后奖励到账，任务标记已完成

---

### 4.3 打卡与家长确认

**功能描述：** 连接孩子行动与家长监督的闭环环节。

**孩子端：**

- 任务卡片上有明显的「打卡」按钮（大按钮，适合儿童点击）。
- 打卡成功显示鼓励文案，如「太棒了！等爸爸妈妈确认吧～」。

**家长端：**

- 「待确认」入口（Tab 角标或首页提示）。
- 待确认列表展示：任务名、打卡时间、确认 / 驳回按钮。
- 确认后播放简单成功提示（文字 + 图标即可，无复杂动画）。

**业务规则：**

- 每日每任务仅可打卡一次。
- 跨日未确认的任务：MVP 策略为**自动作废**或**次日仍可确认**（建议：次日 0 点自动回退为未完成，简化逻辑）。

**验收标准：**

- [ ] 孩子打卡 → 家长可见待确认项
- [ ] 家长确认 → 道具发放、任务完成
- [ ] 同日重复打卡被拦截

---

### 4.4 奖励机制

**功能描述：** 完成任务并经家长确认后，自动发放养成道具。

**道具类型：**

| 道具 | 用途 | 每次使用效果 |
|------|------|--------------|
| 食物 | 喂食 | 饥饿值 +20，成长值 +5 |
| 肥皂 | 洗澡 | 清洁值 +20，成长值 +5 |
| 玩具 | 陪玩 | 心情值 +20，成长值 +5 |

**规则：**

- 道具存储在背包，不过期（MVP）。
- 单次操作消耗 1 个道具。
- 状态值上限 100，超出部分不计入（成长值仍计入）。

**验收标准：**

- [ ] 确认任务后背包数量正确增加
- [ ] 使用道具后背包减少、宠物状态提升

---

### 4.5 宠物养成与升级

**功能描述：** 通过消耗道具提升宠物状态，累计成长值实现等级提升。

**等级设计（MVP）：**

| 等级 | 累计成长值需求 | 形象变化 |
|------|----------------|----------|
| Lv.1 | 0 | 幼崽 |
| Lv.2 | 50 | 幼年 |
| Lv.3 | 120 | 少年 |
| Lv.4 | 220 | 青年 |
| Lv.5 | 350 | 成年 |

**升级反馈：**

- 弹窗提示「小萌升级啦！现在是 Lv.X」。
- 更新宠物形象图片。

**状态衰减（MVP 简化方案）：**

- 每日 0 点将饥饿 / 清洁 / 心情重置为 60（或保持不变），避免过度惩罚，降低开发复杂度。
- 后续版本可改为按小时衰减。

**验收标准：**

- [ ] 成长值累计正确
- [ ] 达到阈值自动升级
- [ ] 升级后形象切换

---

### 4.6 家长设置

**功能描述：** 家长配置每日任务及基础信息。

**功能列表：**

1. **任务管理**
   - 新增任务（名称、图标、奖励类型、奖励数量）
   - 编辑任务
   - 关闭 / 开启任务（软删除，不物理删除）
   - 从预设模板快速添加

2. **宠物设置**
   - 修改宠物名称

3. **角色入口**
   - 切换孩子 / 家长模式

**权限：** 仅家长模式可访问。

**验收标准：**

- [ ] 新增任务次日或当日即时生效（MVP：当日即时生效）
- [ ] 关闭的任务不再出现在孩子今日任务列表
- [ ] 编辑后未打卡任务展示新配置

---

### 4.7 成长记录

**功能描述：** 汇总展示习惯养成与宠物成长数据，增强正向反馈。

**展示内容：**

| 指标 | 说明 |
|------|------|
| 连续打卡天数 | 至少完成 1 项任务并确认即计为打卡日 |
| 累计完成任务数 | 历史确认通过总数 |
| 宠物当前等级 | Lv.X |
| 近 7 日完成情况 | 简易日历或列表（完成 / 未完成） |

**验收标准：**

- [ ] 连续天数计算正确（中断后重新累计）
- [ ] 历史数据与任务记录一致

## 5. 非功能需求

### 5.1 性能

- 首屏加载 < 3 秒（微信开发者工具 / 真机弱网可接受范围内）。
- 操作反馈 < 500ms（本地乐观更新 + 云同步）。

### 5.2 兼容性

- 微信基础库 >= 2.25.0。
- 适配 iPhone 与主流 Android 机型，以 375px 宽度为设计基准。

### 5.3 儿童友好设计

- 主按钮高度 ≥ 88rpx，字号 ≥ 32rpx。
- 色彩明亮、对比清晰，避免过多文字。
- 关键操作有明确图标 + 文字标签。

### 5.4 数据与安全

- MVP 使用微信云开发时，数据按 `openid` 隔离。
- 不涉及支付与敏感个人信息采集。

## 6. 技术方案概要

| 层级 | 方案 |
|------|------|
| 前端 | UniApp（Vue3）编译为微信小程序 |
| 后端 | 优先微信云开发（云数据库 + 云函数）；开发阶段可用本地 mock |
| 状态管理 | Pinia |
| UI | 自定义组件 + 简洁插画资源 |

**本地 mock 策略：** `frontend/mock/` 提供 JSON 数据与 service 层抽象，通过环境变量切换 `mock` / `cloud` 数据源，保证无云环境也能跑通闭环。

---

## 7. MVP 成功标准

第一版在微信开发者工具中可运行，并完成以下闭环：

1. 家长添加任务「刷牙」。
2. 孩子打卡「刷牙」。
3. 家长确认打卡。
4. 孩子获得肥皂 ×1。
5. 孩子使用肥皂为宠物洗澡。
6. 宠物清洁值、成长值提升。
7. 多次完成后宠物升级，成长记录更新。

## 9. 附录：名词表

| 打卡 | 孩子标记某项任务已完成，等待家长确认 |
